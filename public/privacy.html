Privacy Policy

Last updated: March 2026 Peer-to-peer by design No file storage

Velo is built to avoid holding your data. Files travel directly between peers using WebRTC. The only data we touch is the transient signaling required to help peers find each other.

1) The short version

We don’t store your files. We don’t sell your data. We relay only the minimum metadata needed to connect two peers.

2) Data we handle briefly

  • Signaling messages: SDP offers/answers and ICE candidates are relayed through our signaling server so browsers can establish a direct WebRTC path. They are kept in memory only for the duration of setup and then discarded.
  • Room metadata: Active room codes and peer counts live in memory to coordinate who is connected. They disappear when the room ends.

3) File storage

We never store file contents. Transfers move via WebRTC DataChannels straight from sender to receiver.

4) Analytics & cookies

No third-party analytics, no trackers, no cookies beyond what the browser needs to run WebRTC.
